# Getting Started at Duraclim IT

Welcome to the team! This guide will help you get up to speed and ready to contribute effectively to our projects.

<img src="/images/welcome-onboarding.png" alt="Onboarding" className="rounded-lg shadow-md" />

## Your First Day Checklist

<Steps>
  <Step title="Set up your accounts">
    Make sure you have access to all necessary accounts:
    - Company email
    - Slack
    - GitHub
    - Supabase
    - Zapier/Pabbly
    - AWS (if applicable)
  </Step>
  <Step title="Install required software">
    Set up your development environment:
    - [VS Code](https://code.visualstudio.com/) or your preferred IDE
    - [Git](https://git-scm.com/)
    - [Node.js](https://nodejs.org/) (v18+)
    - [Go](https://golang.org/) (v1.19+)
    - [Docker](https://www.docker.com/products/docker-desktop/)
  </Step>
  <Step title="Clone key repositories">
    ```bash
    git clone https://github.com/duraclim/duraclim-calculator.git
    git clone https://github.com/duraclim/go-reschedule-client.git
    git clone https://github.com/duraclim/knowledge-base.git
    ```
  </Step>
  <Step title="Join communication channels">
    Make sure you're in these Slack channels:
    - #it
    - #it-discussions
    - #it-testing
    - #it-alerts
    - #it-asana
  </Step>
</Steps>

## Development Workflow

Our development process follows these general steps:

<Steps>
  <Step title="Pick up a task">
    Tasks are tracked in Asana. Assign yourself to a task or ask your team lead what to work on next.
  </Step>
  <Step title="Create a feature branch">
    Create a branch from main following our naming convention: 
    ```
    feature/short-description-of-task
    ```
  </Step>
  <Step title="Implement and test">
    Develop the feature, writing tests along the way. Follow our coding style guidelines and patterns.
  </Step>
  <Step title="Submit for review">
    Create a pull request and request reviews from relevant team members.
  </Step>
  <Step title="Address feedback">
    Make necessary changes based on PR feedback.
  </Step>
  <Step title="Merge">
    Once approved, your code will be merged into the main branch.
  </Step>
</Steps>
