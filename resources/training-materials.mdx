### First Projects Checklist

Recommended initial projects to familiarize yourself with our systems:

<AccordionGroup>
  <Accordion title="Frontend Developers">
    1. **Calculator UI Enhancement**
       - Add a new feature to the calculator UI
       - Collaborate with a senior developer for guidance
       - Get familiar with our React/Next.js patterns
       
    2. **Component Library Exploration**
       - Review our UI component library
       - Create a new reusable component
       - Add proper documentation and tests
  </Accordion>
  
  <Accordion title="Backend Developers">
    1. **Go API Endpoint**
       - Implement a new API endpoint in one of our Go services
       - Write comprehensive tests
       - Document the API using our standards
       
    2. **Database Migration Task**
       - Create a database migration for a feature
       - Follow our migration guidelines
       - Test the migration in all environments
  </Accordion>
  
  <Accordion title="Full-Stack Developers">
    1. **End-to-End Feature Implementation**
       - Implement a small feature spanning frontend and backend
       - Coordinate with team members as needed
       - Document your implementation decisions
       
    2. **Integration Test Suite**
       - Create integration tests for an existing feature
       - Ensure test coverage for critical paths
       - Set up CI/CD for the tests
  </Accordion>
</AccordionGroup>

- Quarterly learning goals should be set and reviewed with your manager

## Training Content Creation

We encourage team members to create internal training materials:

<AccordionGroup>
  <Accordion title="Training Formats">
    - **Lunch & Learn Sessions:** 30-60 minute presentations during lunch
    - **Hands-on Workshops:** Interactive sessions with practical exercises
    - **Recorded Tutorials:** Screen recordings with narration
    - **Written Guides:** Step-by-step documentation for specific topics
    - **Code Walkthroughs:** Detailed explanations of codebases or features
  </Accordion>
  
  <Accordion title="Content Creation Tools">
    - [Loom](https://www.loom.com/) - For screen recording with audio
    - [Miro](https://miro.com/) - For collaborative diagrams and workflows
    - [Excalidraw](https://excalidraw.com/) - For hand-drawn style diagrams
    - [Google Slides](https://slides.google.com/) - For presentations
    - [VS Code + Live Share](https://code.visualstudio.com/learn/collaboration/live-share) - For code walkthroughs
  </Accordion>
  
  <Accordion title="Content Guidelines">
    - **Be concise:** Focus on the most important concepts
    - **Show, don't tell:** Use examples and demonstrations
    - **Build incrementally:** Start simple and add complexity gradually
    - **Provide context:** Explain why, not just how
    - **Include exercises:** Give learners opportunities to practice
    - **Gather feedback:** Collect feedback to improve future sessions
  </Accordion>
</AccordionGroup>

To propose a new training session, submit the [training creation form](https://forms.duraclim.com/training-creation).
