# Recommended Reading

This document provides a curated collection of books, articles, blog posts, and online resources that are valuable for the Duraclim IT team. These resources cover technical topics, best practices, and industry knowledge relevant to our stack and business domain.

## Technical Books

### Software Development

<CardGroup cols={2}>
  <Card title="Clean Code" icon="code">
    **Author:** Robert C. Martin
    
    Essential principles for writing maintainable, readable code. Particularly relevant for our Go and JavaScript codebases.
    
    **Key topics:** Code clarity, meaningful naming, functions, comments, formatting, error handling
    
    [Amazon Link](https://www.amazon.com/Clean-Code-Handbook-Software-Craftsmanship/dp/0132350882)
  </Card>
  
  <Card title="Designing Data-Intensive Applications" icon="database">
    **Author:** Martin Kleppmann
    
    In-depth guide to data systems and how to build applications that are reliable, scalable, and maintainable.
    
    **Key topics:** Database internals, data models, encoding, distributed systems
    
    [Amazon Link](https://www.amazon.com/Designing-Data-Intensive-Applications-Reliable-Maintainable/dp/1449373321)
  </Card>
  
  <Card title="The Pragmatic Programmer" icon="laptop-code">
    **Authors:** Andrew Hunt & David Thomas
    
    Practical advice for improving your craft as a software developer. Applicable to all our programming environments.
    
    **Key topics:** Code quality, automation, testing, career development
    
    [Amazon Link](https://www.amazon.com/Pragmatic-Programmer-journey-mastery-Anniversary/dp/0135957052)
  </Card>
  
  <Card title="Refactoring" icon="code-compare">
    **Author:** Martin Fowler
    
    Essential guide to improving code quality and maintainability without changing external behavior.
    
    **Key topics:** Code smells, refactoring techniques, testing
    
    [Amazon Link](https://www.amazon.com/Refactoring-Improving-Existing-Addison-Wesley-Signature/dp/0134757599)
  </Card>
</CardGroup>

### Technology Specific

<CardGroup cols={2}>
  <Card title="Go Programming Language" icon="golang">
    **Authors:** Alan A. A. Donovan & Brian W. Kernighan
    
    The definitive guide to the Go programming language. Essential reading for our backend developers.
    
    **Key topics:** Go fundamentals, concurrency, interfaces, testing
    
    [Amazon Link](https://www.amazon.com/Programming-Language-Addison-Wesley-Professional-Computing/dp/0134190440)
  </Card>
  
  <Card title="Learning React" icon="react">
    **Authors:** Alex Banks & Eve Porcello
    
    Comprehensive guide to React development. Great for our frontend team.
    
    **Key topics:** React fundamentals, hooks, state management, testing
    
    [Amazon Link](https://www.amazon.com/Learning-React-Modern-Patterns-Developing/dp/1492051721)
  </Card>
  
  <Card title="Fullstack Next.js" icon="react">
    **Author:** Leigh Halliday
    
    In-depth guide to building applications with Next.js. Highly relevant for our web application development.
    
    **Key topics:** Next.js, API routes, SSR, ISR, authentication
    
    [Publisher Link](https://www.newline.co/fullstack-next-js)
  </Card>
  
  <Card title="Database Internals" icon="database">
    **Author:** Alex Petrov
    
    Deep dive into database design and implementation. Great for understanding our PostgreSQL database better.
    
    **Key topics:** Storage engines, distributed systems, replication
    
    [Amazon Link](https://www.amazon.com/Database-Internals-Deep-Distributed-Systems/dp/1492040347)
  </Card>
</CardGroup>

### Architecture & Design

<CardGroup cols={2}>
  <Card title="Clean Architecture" icon="sitemap">
    **Author:** Robert C. Martin
    
    Principles of software architecture that are applicable to our projects.
    
    **Key topics:** SOLID principles, component boundaries, microservices
    
    [Amazon Link](https://www.amazon.com/Clean-Architecture-Craftsmans-Software-Structure/dp/0134494164)
  </Card>
  
  <Card title="Building Microservices" icon="network-wired">
    **Author:** Sam Newman
    
    Comprehensive guide to building microservice architectures. Relevant for our service architecture.
    
    **Key topics:** Service design, deployment, testing, monitoring
    
    [Amazon Link](https://www.amazon.com/Building-Microservices-Designing-Fine-Grained-Systems/dp/1492034029)
  </Card>
  <Card title="Domain-Driven Design" icon="diagram-project">
    **Author:** Eric Evans
    
    How to connect complex domain models to software implementation. Valuable for our business logic implementation.
    
    **Key topics:** Domain modeling, bounded contexts, ubiquitous language
    
    [Amazon Link](https://www.amazon.com/Domain-Driven-Design-Tackling-Complexity-Software/dp/0321125215)
  </Card>
  
  <Card title="Patterns of Enterprise Application Architecture" icon="cubes">
    **Author:** Martin Fowler
    
    Catalog of enterprise application architecture patterns. Useful reference for our application design.
    
    **Key topics:** Domain logic, data source architecture, web presentation
    
    [Amazon Link](https://www.amazon.com/Patterns-Enterprise-Application-Architecture-Martin/dp/0321127420)
  </Card>
</CardGroup>

## How to Contribute

Have a resource to recommend? Follow these steps to add it to our collection:

<Steps>
  <Step title="Review existing resources">
    Check if the resource is already included in this document.
  </Step>
  
  <Step title="Evaluate the resource">
    Ensure it's high-quality, relevant to our work, and provides value.
  </Step>
  
  <Step title="Submit a pull request">
    Fork the knowledge base repository, add your resource to the appropriate section, and submit a PR.
  </Step>
  
  <Step title="Include context">
    Explain why you're recommending this resource and how it relates to our work.
  </Step>
</Steps>

Or simply share it in the #it-reading-list Slack channel for discussion.
