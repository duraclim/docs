## Setting Up Your Development Environment

<Steps>
  <Step title="Install essential software">
    Download and install these core tools:
    - Visual Studio Code
    - Git
    - Docker Desktop
    - Node.js (LTS version)
    - Go (if working on backend services)
    - Python (if working on automation scripts)
  </Step>
  
  <Step title="Configure VS Code">
    Install required extensions:
    ```
    code --install-extension dbaeumer.vscode-eslint
    code --install-extension esbenp.prettier-vscode
    code --install-extension golang.go
    code --install-extension ms-python.python
    code --install-extension bradlc.vscode-tailwindcss
    ```
    
    Copy our standard VS Code settings from [this gist](https://gist.github.com/duraclim/vscode-settings).
  </Step>
  
  <Step title="Set up Git">
    Configure Git with your user information:
    ```bash
    git config --global user.name "Your Name"
    git config --global user.email "your.email@duraclim.com"
    ```
    
    Generate and add an SSH key to your GitHub account following [GitHub's documentation](https://docs.github.com/en/authentication/connecting-to-github-with-ssh/generating-a-new-ssh-key-and-adding-it-to-the-ssh-agent).
  </Step>
  
  <Step title="Clone repositories">
    Clone the repositories you'll be working with:
    ```bash
    git clone git@github.com:duraclim/duraclim-calculator.git
    git clone git@github.com:duraclim/go-reschedule-client.git
    git clone git@github.com:duraclim/knowledge-base.git
    ```
  </Step>
  
  <Step title="Set up authentication">
    Request access to:
    - Supabase project
    - AWS IAM user
    - 1Password vault
    - Slack workspace
    
    Follow the onboarding emails you receive for each service.
  </Step>
</Steps>

## Tool Access Request Process

To request access to any of the tools mentioned above:

1. Fill out the [Access Request Form](https://forms.duraclim.com/access-request)
2. Include the specific tool and access level needed
3. Provide justification for the request
4. Your manager will approve the request
5. The IT team will grant access and notify you

For urgent access needs, contact the IT team directly via Slack in the #it-discussions channel.

## Tool Policies

<AccordionGroup>
  <Accordion title="Personal vs. Company Accounts">
    Always use your Duraclim email account for all work-related tools. Never use personal accounts for company work, as this creates security risks and access management issues.
  </Accordion>
  
  <Accordion title="Data Security">
    Respect the confidentiality of company data. Do not share access credentials, export sensitive data to unmanaged tools, or bypass security controls.
  </Accordion>
  
  <Accordion title="Tool Suggestions">
    We welcome suggestions for new tools that could improve our workflow. To propose a new tool:
    
    1. Research the tool and identify clear benefits
    2. Check for any integration capabilities with our existing stack
    3. Submit a proposal in the #it-discussions channel
    4. IT will evaluate the suggestion and respond within 5 business days
  </Accordion>
  
  <Accordion title="Departing Employee Process">
    When leaving the company:
    
    1. Transfer ownership of any documents or projects to your manager
    2. Do not delete work-related data from company tools
    3. Return all access credentials and licensed software
    4. IT will systematically revoke access to all systems
  </Accordion>
</AccordionGroup>
